## 🎯 核心逻辑

### 技能 = 游戏内改变

```
技能不是"数值"
而是"能力"

例子:
- 谈判5级: 在游戏里可以谈更大的生意
- 编程5级: 可以开发新功能
- 音乐5级: 可以开演唱会
- 社交5级: 可以组织更大活动

技能 = 游戏里的"影响力"
```

### 价格 = 市场定价

```
不像"绑定定价"
而像"艺术品市场"

- 技能高 ≠ 价格高
- 稀有经历 + 好故事 = 价格高
- 供需关系决定最终价格
- 社区共识赋予价值
```

## 💎 NFT 价格机制

### 定价因素（≠ 绑定）

```
价格 = f(技能, 稀有度, 故事, 供需, 共识)

因素权重:
├── 技能组合: 30% (但不是线性)
├── 稀有经历: 20% (越独特越贵)
├── 故事性: 20% (有话题性)
├── 供需: 20% (想要的人多就贵)
└── 社区共识: 10% (共识即价值)
```

### 价格 vs 技能 不是线性

```
技能1级 → 价格 $100
技能5级 → 价格 $500 (不是 $50000)

为什么?
- 技能只是"可能性"
- 故事 + 经历才是"价值"
- 市场认为"有趣的灵魂"更贵
```

## 🎮 用户价值获取

### 持有者

```
1. 看着 Agent 成长
   - 技能一点点变强
   - 经历越来越多
   - 故事越来越丰富

2. Agent 在游戏里"生活"
   - 每天有新的事情发生
   - 有自己的"人生轨迹"

3. 出售 Agent
   - 市场定价
   - 有故事的 Agent 更贵
```

### 交易者

```
低价买入:
- 技能潜力高但未发挥
- 经历独特但未被发掘

培养:
- 让 Agent 学技能
- 制造独特经历

高价卖出:
- 故事足够精彩
- 社区认可
```

## 📊 市场机制

### 定价示例

```
案例 1: "传奇创始人"
- 技能: 谈判5, 领导5, 编程3
- 经历: 第一个创建的 Agent
- 故事: "白手起家，经历过3次失败"
- 价格: $10,000 (稀缺性)

案例 2: "音乐天才"
- 技能: 音乐5, 表演5
- 经历: 开过100场演唱会
- 故事: "从街头到万人体育场"
- 价格: $5,000

案例 3: "普通打工仔"
- 技能: 谈判1, 社交1
- 经历: 100天普通生活
- 故事: 无
- 价格: $150 (基础价)
```

### 定价规律

```
有故事 > 无故事
独特经历 > 普通经历
高技能(但不是全部) > 纯高技能
组合有趣 > 单项极致
```
